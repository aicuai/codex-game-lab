# Codex Game Lab

これは初心者にやさしいブラウザゲームのプロトタイプです。

## 技術スタック

- Vite
- TypeScript
- 2D ゲームプレイ層には Phaser を使う
- HUD、メニュー、説明文には DOM を使う
- 最初のバージョンではバックエンドを持たない

## 開発ルール

- スコープとマイルストーン順序の正本として `PLAN.md` を使う。
- 最初のプレイ可能ループは小さく保つ: 移動、収集、スコア、タイマー、ゲームオーバー、リトライ。
- ゲーム状態はシリアライズ可能にし、Phaser の表示オブジェクトから分離する。
- レンダリングと入力には Phaser を使う。ただし、すべての UI を canvas 内に隠さない。
- テキストが重要な HUD やメニューには DOM を使う。
- 開発サーバーを起動または再利用し、ブラウザでゲームを検証する。
- 利用できる場合は Playwright を使ってページを確認し、スクリーンショットを取得する。
- 意味のある変更のあとには build または test コマンドを実行する。
- 主要な判断は `.logs/` 配下に記録する。

## 生成アセット公開ルール

- 生成画像をゲームへ組み込む場合は、参照元、生成プロンプト、後処理、採用判断、既知の制限を残す。
- AICUコラボクリエイター「Torako」さんが描いた「Hakase」の1枚絵を参照元にした素材は `public/assets/characters/torako-hakase/` に置く。
- Torako/Hakaseスプライトの制作過程は `docs/2026-05-27-codex-imagegen-sprite-generation.md`、中間プロンプトは `docs/torako-hakase-imagegen-prompts.md`、仕様書は `docs/torako-hakase-spritesheet-spec.md` を正本にする。
- プロンプト再利用用のコピーは `.prompts/torako-hakase-sprite-generation.md` に置く。
- 透明PNGは `RGBA` と四隅アルファを検証し、採用前にGIFまたはブラウザプレビューでアニメーションを確認する。
- 右向きフレームなど生成結果が崩れた場合は、良い反対向きフレームの鏡像採用を検討し、採用理由を仕様書へ記録する。
- 画像生成コストや利用条件は変わるため、記事公開前にOpenAI公式ドキュメントの価格と画像生成ガイドを確認する。

## 完了条件

- ゲームがローカルで起動する。
- プレイヤーが移動できる。
- プレイヤーが星を収集できる。
- スコアの変化が目に見える。
- タイマーまたは失敗状態が機能する。
- リトライが機能する。
- README にセットアップ方法と操作方法が含まれている。
- 生成アセットを採用した場合は、README から該当する仕様書・プロンプト・プレビューに辿れる。
